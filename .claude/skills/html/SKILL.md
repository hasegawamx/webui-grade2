# HTML ベストプラクティス（最低限）

このスキルは「壊れていない HTML」ではなく「意図が伝わる HTML」を書くための最低限の指針。迷ったらここに戻る。

## セマンティクスを優先する

意味のあるタグが使える場面で `<div>` / `<span>` を選ばない。意味のあるタグは、スクリーンリーダー・検索エンジン・開発者ツールの全員に「これは何か」を一度で伝える。

- ナビゲーション → `<nav>`
- 本文の主要コンテンツ → `<main>`（1 ページに 1 つ）
- 記事・投稿単位 → `<article>`
- 章・セクション → `<section>`（見出しとセットで）
- 補足情報・サイドバー → `<aside>`
- ヘッダー／フッター → `<header>` / `<footer>`
- ボタン挙動 → `<button>`（`<div onclick>` は禁止、キーボード操作とフォーカスを失う）
- リンク遷移 → `<a href>`（`<button>` で遷移しない）

見出しは `h1` → `h2` → `h3` の階層を飛ばさない。レベルは「見た目の大きさ」ではなく「情報の入れ子構造」で決める。

## アクセシビリティの最低ライン

以下が満たされていないマークアップはレビューで差し戻すこと。

- `<img>` には必ず `alt`。装飾画像なら `alt=""`（属性ごと省略しない）。
- フォーム入力には `<label for="id">` を明示的に結びつける。`placeholder` はラベルの代わりにならない。
- ボタンやアイコンだけのコントロールには `aria-label` を付ける。
- `<html lang="ja">` のように言語を指定する。
- タブ順序が DOM 順序と一致するように書く。`tabindex` の正の値はほぼ常にバグ。
- コントラストや色だけで情報を伝えない（色 + テキスト or アイコン）。

## 文書構造

```html
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    <title>ページの内容を表すタイトル</title>
  </head>
  <body>
    <header>...</header>
    <main>...</main>
    <footer>...</footer>
  </body>
</html>
```

`viewport` の meta はモバイル対応の前提。忘れると CSS のレスポンシブ指定が無力化する。

## やりがちな失敗

- `<b>` / `<i>` を強調のために使う → 強調なら `<strong>` / `<em>`、ただの太字・斜体なら CSS で。
- `<br>` を連打して余白を作る → 余白は CSS の margin / padding。
- テーブルレイアウト → `<table>` は表データ専用。レイアウトは grid / flex。
- インライン `style` 属性 → 例外的な一発限りの調整以外は CSS ファイルへ。
- クリックできる `<div>` → 必ず `<button>` か `<a>` に置き換える。
