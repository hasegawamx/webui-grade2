# CSS ベストプラクティス（最低限）

CSS は「動く」と「壊れない」の距離が遠い。このスキルは、後から触る自分と他人を助けるための最低限の指針。

## レイアウトはモダンな仕組みに寄せる

- 1 次元の並び（横並び／縦並び）→ Flexbox
- 2 次元の格子 → CSS Grid
- `float` / `position: absolute` をレイアウト主手段にしない（回り込みやオーバーレイなど本来の用途に限る）
- 固定幅より `min()` / `max()` / `clamp()` と相対単位（`rem` / `%` / `fr`）を優先する

## レスポンシブはモバイルファースト

ベースをモバイル向けに書き、`min-width` のメディアクエリで拡張する。逆方向より上書きが少なく、バグが減る。

```css
.card { padding: 1rem; }

@media (min-width: 768px) {
  .card { padding: 2rem; }
}
```

## 詳細度とセレクタ

- クラスベースで書く。ID セレクタと要素型セレクタの多段ネストはなるべく避ける。
- 詳細度を上げて勝とうとせず、セレクタを素直に保つ。勝ちたくなった時点で設計が歪んでいる。
- `!important` は最終手段。サードパーティ CSS を上書きする等の明確な理由がないなら書かない。理由があるならコメントで残す。

## 設計の型を 1 つ選ぶ

プロジェクト内で命名とスコープの方針を 1 つに揃える。混在は事故のもと。

- BEM（`.block__element--modifier`）
- Utility-first（Tailwind 等）
- CSS Modules / scoped styles
- 単一ファイル component の `<style scoped>`

既存コードに方針があるなら、好みより一貫性を優先する。

## カスタムプロパティで「設定」を集約

色・間隔・フォントなど繰り返す値は CSS 変数にする。テーマ切替・ダークモード対応もこれが前提。

```css
:root {
  --color-text: #1a1a1a;
  --color-bg: #fff;
  --space-1: 0.25rem;
  --space-2: 0.5rem;
}

@media (prefers-color-scheme: dark) {
  :root {
    --color-text: #f0f0f0;
    --color-bg: #111;
  }
}
```

## ボックスモデルと単位

- グローバルに `* { box-sizing: border-box; }` を置く（多くのリセット CSS はこれを含む）。
- フォントサイズは `rem`、コンポーネント内の余白は `em` or `rem`、`px` は 1px の枠線など「本当に 1px 必要な」場面に限定。
- 行の高さは単位なし（`line-height: 1.5`）で指定するとフォント継承時に事故りにくい。

## やりがちな失敗

- 巨大な「全部入り」セレクタ（`.page .main .section .card .title span`）
- 要素の並びを `margin-left: -10px` のような負マージンで帳尻合わせ
- `z-index: 9999` の応酬（スタッキングコンテキストを理解する）
- 任意の色をその場でハードコード → カスタムプロパティに寄せる
- アニメーションで `width` / `height` / `top` / `left` を変える（`transform` / `opacity` が GPU 合成で軽い）

## チェックリスト

1. 新しい色・間隔を加えたなら、それは変数化すべきか
2. `!important` を足したなら、その理由をコメントに書いたか
3. 詳細度で勝負せず、クラス設計で解決できないか
4. モバイル幅で壊れていないか（DevTools で確認）
5. ダークモードや prefers-reduced-motion を壊していないか
